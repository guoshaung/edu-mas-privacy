隐私保护算法

这一块把你的隐私保护思路按“核心创新 + 支撑机制 + 评价函数”拆开讲清楚。
Privacy Side

1. SRPG 双流融合机制

把学生原始输入拆成两条流：一条承担个体敏感特征的受控表达，一条承担教学任务真正需要的逻辑语义。
去敏流 逻辑流 差分隐私 多智能体受控读取
  • 去敏流保留桶化后的认知风格、压力水平、行为模式和支持强度。
  • 逻辑流保留当前主题、问题摘要、语义焦点和辅导目标。
  • 最终不同 Agent 读取不同组合，而不是统一读取一整份原始输入。

2. 动态隐私信任评估函数

\[ T_{p}(t)=\alpha T_{\text{base}}-\beta D_{t}-\gamma R_{\text{gnn}}-\lambda E_{t} \]
  • \(T_{\text{base}}\)：初始隐私信任基线
  • \(D_{t}\)：累计交互时长或会话持续时间
  • \(R_{\text{gnn}}\)：FedGNN 计算得到的风险分数
  • \(E_{t}\)：累计暴露强度，例如身份、行为、心理、画像字段的暴露量
动态评估 FedGNN 驱动 控制开放粒度

版权保护算法

版权侧不强调单一检测模块，而是强调对多智能体资源流转过程的动态评估与治理。
Copyright Side

3. 动态版权合规评估函数

\[ C_{r}(t)=\mu R_{c}+\nu R_{a}+\xi R_{p}+\rho R_{e}+\sigma R_{i} \]
  • \(R_{c}\)：版权资源敏感度
  • \(R_{a}\)：主体授权匹配风险
  • \(R_{p}\)：多智能体传播风险
  • \(R_{e}\)：内容暴露风险
  • \(R_{i}\)：迭代累积风险
动态版权风险 内容交付粒度控制 AG4 检索治理

4. 支撑机制

  • ABAC：控制不同主体、不同阶段、不同资源类型的访问权限。
  • FedGNN：根据通信时序、频率、载荷体积和语义分布给出风险输入。
  • 污点分析：跟踪版权标签在 AG4 → AG3 → 学生端之间的流动。
  • 水印与溯源：用于后续责任追踪，不是当前主线创新。
ABAC FedGNN 污点传播 溯源追踪

联合治理策略

隐私保护和版权保护不是各自独立工作，而是最后在治理层一起决定系统交付方式。
Joint Decision
高 T_p 低 C_r 正常放行，允许完整教学交付
低 T_p 低 C_r 增强脱敏后放行，仅提供必要个体特征
高 T_p 高 C_r 不返原文，只给摘要或替代生成内容
低 T_p 高 C_r 阻断或最小必要返回，并记录治理日志

一句话概括

你的系统不是简单“加一个隐私模块”和“加一个版权模块”，而是在多智能体系统之上增加了一个联合治理层：
\[ \mathrm{Joint\ Policy}=F\!\left(T_{p}(t),\,C_{r}(t),\,\mathrm{phase},\,\mathrm{role},\,\mathrm{resource}\right) \]
也就是说，系统最终是根据隐私信任、版权风险、当前阶段、主体角色和资源等级共同决定怎么交付内容。

变量说明与答辩提示

这部分更偏“讲解辅助”，方便你在汇报时快速切换话术。
Presentation Notes
你可以怎么讲 SRPG
SRPG 的意义不在于机械拆分，而在于把“教学任务需要的语义”与“学生个体敏感性”分开治理，并让不同 Agent 读取不同粒度的信息。
你可以怎么讲 T_p(t)
T_p(t) 不是一个静态常数，而是随着交互时长、行为模式变化和结构化风险变化动态更新的隐私信任值。
你可以怎么讲 C_r(t)
C_r(t) 不是只看资源本身，而是看资源在多智能体系统中的传播、暴露和多轮交互累积后的合规风险。
你可以怎么讲联合治理
隐私保护面向学生数据，版权保护面向教师资源，二者在治理层统一决策，保证系统既能教、又不会过度暴露。
